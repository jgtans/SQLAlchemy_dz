# SQLAlchemy_dz
Домашнее задание: SQLAlchemy — часть 2 (Pydantic + DummyJSON)

Короткое описание
-----------------
Сервис для загрузки и хранения товаров из DummyJSON API с использованием SQLAlchemy ORM и Pydantic для валидации данных. Проект сохраняет данные в SQLite-базу `dummyjson_shop.sqlite3`.

Особенности
-----------
- Загружает товары из DummyJSON (https://dummyjson.com).
- Сохраняет данные в SQLite через SQLAlchemy ORM.
- Использует Pydantic для валидации/DTO (ProductDTO).
- Предотвращает дубликаты брендов и категорий функцией `get_or_create()`.

Требования
---------
- Python 3.8+
- зависимости перечислены в `requirements.txt`

Установка
---------
Рекомендуется создать виртуальное окружение:

```bash
git clone https://github.com/jgtans/SQLAlchemy_dz.git
cd SQLAlchemy_dz
python -m venv .venv
# на Unix/macOS
source .venv/bin/activate
# на Windows
.venv\Scripts\activate

pip install -r requirements.txt
```

Запуск
------
Запустить основную программу командой:

```bash
python main.py
```

После запуска будет создан(а) файл `dummyjson_shop.sqlite3` в корне проекта и туда будут добавлены записи.

Если нужно начать с чистой БД — удалите `dummyjson_shop.sqlite3` и запустите снова.

Структура проекта
-----------------
- db.py — настройка подключения к БД (engine, Base, Session)
- models.py — модели БД (Brand, Category, Product, Review) и DTO (ProductDTO)
- dummyjson_loader.py — функции для загрузки/парсинга данных из DummyJSON API
- main.py — точка входа приложения
- requirements.txt — зависимости
- dummyjson_shop.sqlite3 — SQLite база (создается автоматически)

База данных
----------
- Файл: `dummyjson_shop.sqlite3`
- Для очистки данных просто удалите файл и перезапустите загрузчик.
- Функция `get_or_create()` предотвращает дублирование брендов и категорий при повторных запусках.
